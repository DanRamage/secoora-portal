    Description:
    Abstract:
    This data set contains OCS block outlines in ArcGIS shape file format for
    the BOEM Atlantic Region.  OCS blocks are used to define small geographic
    areas within an Official Protraction Diagram (OPD) for leasing and administrative
    purposes.  These blocks have been clipped along the Submerged Lands Act
    (SLA) boundary and along the Continental Shelf Boundaries.  Additional details
    are available from:  http://www.boem.gov/uploadedFiles/BOEM/Oil_and_Gas_Energy_Program/Mapping_and_Data/99-0006.pdfBecause GIS projection and topology functions can change or generalize
    coordinates, and because shapefiles can not represent true arcs, these
    GIS files are considered to be approximate and are NOT an OFFICIAL record
    for the exact block coordinates or areas. The Official Protraction Diagrams
    (OPDs) and Supplemental Official Block Diagrams (SOBDs) serve as the legal
    definition for BOEM offshore boundary coordinates and area descriptions.
     
    Purpose:
    These block polygons and other cadastre information the BOEM produces are
    generated in accordance with 30 Code of Federal Regulations (CFR) 256.8
    to support Federal land ownership and mineral resource management.
     
    Supplemental_Information:
    The BOEM is making its existing cadastre data available at
    http://www.boem.gov/Oil-and-Gas-Energy-Program/Mapping-and-Data/Index.aspx.  These data include Official
    Protraction Diagrams (OPDs), and Supplemental Official OCS Block
    Diagrams (SOBDs) in pdf format, and digital spatial data in ArcGIS
    shapefile format.

    Data_Quality_Information:
    Logical_Consistency_Report:
    The file is believed to be free of topologic errors.
     
    Completeness_Report:
    All federal blocks from the Submerged Lands Act Boundary (3 mile line)
    out to the Continental Shelf Boundary or Limit of Protraction are included.
     
    Positional_Accuracy:
    Horizontal_Positional_Accuracy:
    Horizontal_Positional_Accuracy_Report:
    Coordinates values have been computed to .00000001 of a degree, but this shapefile is considered to be approximate.  Where differences occur between the shapefile and the OPD and SOBD diagrams, the diagrams take precidence.
    This shapefile was created by extracting the X-Y UTM coordinates from the TIMS database.  In order to create this Regional shapefile, which spans multiple UTM zones, all coordinates were re-projected to a single UTM zone (Zone 18).  This file was then re-projected to geographic for distribution purposes.  This resulted in some distortion (up to 0.009 meters) at the far East and West edges of the shapefile.

    Process_Step:
    Process_Description:
    In house programs were used to protract the blocks on a UTM grid
    for the offshore areas, and to intersect those blocks with various
    boundaries including the SLA and CSB boundaries.  The resulting
    points were then stored in the TIMS database.  Other in-house
    program were used to create an ArcInfo Coverage from
    the data stored in the TIMS database.  That coverage was
    converted into a shapefile using ArcGIS, and any portions of blocks
    landward of the SLA boundary or seaward of the CSB boundary were
    removed.  The shapefile was further "disolved" to remove any other
    internal boundary lines, leaving only the block outlines.
